# 不相交集合数据结构 (Disjoint-Set Data Structure)

> 📘 _《算法导论》第21章学习指南_

## 🎯 1. 简介

**不相交集合数据结构**(Disjoint-Set Data Structure)，也被称为**并查集**(Union-Find Set)，是一种用于处理不相交集合合并与查询问题的树型数据结构。它支持三种基本操作：

1. **MAKE-SET(x)** - 创建一个只包含元素x的新集合
2. **UNION(x, y)** - 将包含元素x和y的两个集合合并为一个集合
3. **FIND-SET(x)** - 返回包含元素x的集合的代表元素（通常是根节点）

### 1.1 历史背景

不相交集合数据结构的概念最早由 Bernard A. Galler 和 Michael J. Fischer 在1964年提出。他们不仅提出了这种数据结构，还对其进行了详细的分析。后来，Robert Tarjan 在1975年证明了该数据结构在特定操作序列下的时间复杂度上界。

### 1.2 应用场景

不相交集合数据结构在计算机科学中有广泛的应用：

- **图论算法**：Kruskal最小生成树算法
- **网络连通性**：判断网络中节点间的连通性
- **图像处理**：连通区域标记
- **编译器设计**：寄存器分配中的变量冲突检测
- **社交网络分析**：朋友圈识别

## 📚 2. 基本概念

### 2.1 不相交集合的性质

不相交集合具有以下重要性质：

1. **互斥性**：任意两个集合要么完全相同，要么没有任何公共元素
2. **完整性**：所有元素都必须属于某个集合
3. **代表性**：每个集合都有一个代表元素，通常选择根节点作为代表

### 2.2 集合的表示方法

不相交集合通常用树的形式来表示：

- 每个集合用一棵有根树表示
- 树中的每个节点存储集合中的一个元素
- 每个节点还有一个指向其父节点的指针
- 根节点是集合的代表元素，其父节点指向自身

例如，假设有三个集合：
- S₁ = {0, 1, 2}
- S₂ = {3, 4}
- S₃ = {5, 6, 7, 8}

可以用如下树结构表示：

```
S₁:     S₂:     S₃:
  0       3       5
 / \       \     /|\
1   2       4   6 7 8
```

### 2.3 链接表示法

为了有效地实现不相交集合，我们使用链接表示法(linked-list representation)或者更常用的树表示法(tree representation)。在我们的实现中，使用一个数组来表示每个元素的父节点，这种方法叫做**双亲表示法**。

对于每个元素i，我们维护：
- `parent[i]`：指向元素i的父节点
- 当`parent[i] = i`时，表示元素i是根节点，也是其所在集合的代表元素

## 🔧 3. 基本操作详解

### 3.1 MAKE-SET操作

MAKE-SET(x)操作创建一个只包含元素x的新集合。这是最简单的操作：

```cpp
void makeSet(int x) {
    parent[x] = x;
    rank[x] = 0;
}
```

执行MAKE-SET(x)后，元素x将成为一个新集合的唯一成员，同时也是该集合的代表元素。

#### 3.1.1 操作步骤

1. 设置元素x的父节点为自身：`parent[x] = x`
2. 初始化元素x的秩为0：`rank[x] = 0`

这个操作的时间复杂度是O(1)。

### 3.2 FIND-SET操作

FIND-SET(x)操作返回包含元素x的集合的代表元素。实现时，沿着父节点指针向上查找，直到找到根节点：

```cpp
int findSet(int x) {
    if (parent[x] != x) {
        return findSet(parent[x]);
    }
    return parent[x];
}
```

#### 3.2.1 操作原理

FIND-SET操作通过跟踪父节点指针向上移动，直到到达根节点。根节点的特征是其父节点指向自身。

#### 3.2.2 示例演示

假设我们有以下集合结构：
```
    1
   / \
  2   3
      |
      4
```

执行FIND-SET(4)的过程：
1. 从节点4开始，parent[4] = 3 ≠ 4，继续查找parent[3]
2. parent[3] = 1 ≠ 3，继续查找parent[1]
3. parent[1] = 1 = 1，找到根节点，返回1

### 3.3 UNION操作

UNION(x, y)操作将包含元素x和y的两个集合合并。首先需要找到两个元素所在集合的代表元素，然后将其中一个树的根节点作为另一个树的子节点：

```cpp
void unionSets(int x, int y) {
    int rootX = findSet(x);
    int rootY = findSet(y);
    
    if (rootX != rootY) {
        parent[rootY] = rootX;
    }
}
```

#### 3.3.1 操作步骤

1. 使用FIND-SET操作找到x和y所在集合的代表元素rootX和rootY
2. 如果rootX ≠ rootY，说明x和y属于不同集合，需要合并
3. 将其中一个根节点设置为另一个根节点的子节点

#### 3.3.2 示例演示

假设我们有两个集合：
```
集合A:   集合B:
  1        5
 / \      / \
2   3    6   7
    |        
    4        
```

执行UNION(1, 5)后：
```
    1
   /|\
  2 3 5
  | | |\
  4 6 7
```

## ⚡ 4. 优化策略

虽然基本操作已经可以工作，但通过两种优化策略可以显著提高性能：

### 4.1 按秩合并 (Union by Rank)

在按秩合并中，我们总是将秩较小的树连接到秩较大的树下。这里的"秩"可以看作是树的高度上界。

```cpp
void unionSets(int x, int y) {
    int rootX = findSet(x);
    int rootY = findSet(y);
    
    if (rootX != rootY) {
        if (rank[rootX] < rank[rootY]) {
            parent[rootX] = rootY;
        } else if (rank[rootX] > rank[rootY]) {
            parent[rootY] = rootX;
        } else {
            parent[rootY] = rootX;
            rank[rootX]++;
        }
    }
}
```

#### 4.1.1 秩的含义

在按秩合并中，秩(rank)表示树的高度上界。初始时每个节点的秩为0，随着合并操作的进行，秩可能增加。

#### 4.1.2 合并规则

1. 如果两棵树的秩不同，将秩较小的树连接到秩较大的树下
2. 如果两棵树的秩相同，任选其一作为根，并将其秩加1

#### 4.1.3 数学分析

通过按秩合并，可以保证任何秩为r的节点，其子树至少包含2^r个节点。因此，含有n个节点的树的高度至多为log₂(n)。

### 4.2 路径压缩 (Path Compression)

在路径压缩中，我们在执行FIND-SET操作时，将查找路径上的每个节点直接连接到根节点：

```cpp
int findSet(int x) {
    if (parent[x] != x) {
        parent[x] = findSet(parent[x]);  // 递归进行路径压缩
    }
    return parent[x];
}
```

#### 4.2.1 压缩原理

路径压缩在FIND-SET操作期间工作。当我们从一个节点x找到根节点r时，我们将x到r路径上的每个节点直接连接到根节点。

#### 4.2.2 示例演示

假设我们有以下树结构：
```
      1
     / \
    2   3
   /   / \
  4   5   6
 /
7
```

执行FIND-SET(7)之前：
```
      1
     / \
    2   3
   /   / \
  4   5   6
 /
7
```

执行FIND-SET(7)之后，路径上的节点(7,4,2)都直接连接到根节点1：
```
      1
   / /|\ \
  2 3 4 5 6
  |   |
  7   7
```

### 4.3 两种优化结合使用

当同时使用按秩合并和路径压缩时，可以获得最佳的时间复杂度。需要注意的是，路径压缩会改变树的结构，可能使得按秩合并中使用的秩不再是准确的高度上界。但在实际应用中，这种不准确性不会影响算法的正确性。

## 📊 5. 时间复杂度分析

### 5.1 单独使用一种优化

如果只使用按秩合并或只使用路径压缩，各操作的时间复杂度为：

- MAKE-SET: O(1)
- UNION: O(log n)
- FIND-SET: O(log n)

### 5.2 同时使用两种优化

当同时使用按秩合并和路径压缩时，摊还时间复杂度为：

- MAKE-SET: O(1)
- UNION: O(α(n))
- FIND-SET: O(α(n))

其中α(n)是反Ackermann函数，增长极其缓慢。对于任何实际应用中的n值，α(n)都不会超过5。

### 5.3 Ackermann函数与反Ackermann函数

Ackermann函数A(m,n)定义如下：
- A(0,n) = n+1
- A(m,0) = A(m-1,1) (当m>0)
- A(m,n) = A(m-1,A(m,n-1)) (当m>0,n>0)

反Ackermann函数α(n)定义为使得A(i,i)≥n的最小整数i。

一些典型的值：
- α(1) = 1
- α(3) = 2
- α(7) = 3
- α(2^16-1) = 4
- α(2^65536-1) = 5

### 5.4 m次操作序列的复杂度

如果我们执行m次MAKE-SET、UNION和FIND-SET操作序列（其中有n次MAKE-SET操作），则总时间复杂度为：

O(m·α(n))

这几乎是线性的复杂度，非常高效。

## 💡 7. 实现细节

### 7.1 数据结构表示

我们使用两个数组来表示不相交集合：

1. `parent[]`数组：parent[i]表示元素i的父节点
2. `rank[]`数组：rank[i]表示以i为根的树的秩（高度上界）

### 7.2 路径压缩的实现技巧

路径压缩的递归实现简洁明了，但在某些语言中可能导致栈溢出。也可以使用迭代方式实现：

```cpp
int findSetIterative(int x) {
    int root = x;
    // 找到根节点
    while (parent[root] != root) {
        root = parent[root];
    }
    
    // 路径压缩：将路径上的所有节点直接连接到根节点
    while (parent[x] != x) {
        int next = parent[x];
        parent[x] = root;
        x = next;
    }
    
    return root;
}
```

### 7.3 按秩合并的变体

除了使用秩之外，还可以使用节点数作为合并的依据：

```cpp
void unionBySize(int x, int y) {
    int rootX = findSet(x);
    int rootY = findSet(y);
    
    if (rootX != rootY) {
        if (size[rootX] < size[rootY]) {
            parent[rootX] = rootY;
            size[rootY] += size[rootX];
        } else {
            parent[rootY] = rootX;
            size[rootX] += size[rootY];
        }
    }
}
```

这种方式称为按大小合并(Union by Size)。

## 🎯 8. 实际应用案例

### 8.1 Kruskal最小生成树算法

Kruskal算法使用不相交集合来检测添加边是否会形成环路：

```cpp
// Kruskal算法的核心部分
sort(edges.begin(), edges.end());  // 按权重排序边
DisjointSet ds(numVertices);

for (auto& edge : edges) {
    int u = edge.source;
    int v = edge.destination;
    
    // 如果两个顶点不属于同一集合，则添加这条边不会形成环路
    if (ds.findSet(u) != ds.findSet(v)) {
        ds.unionSets(u, v);
        mst.addEdge(edge);
    }
}
```

#### 8.1.1 算法步骤

1. 将所有边按照权重从小到大排序
2. 初始化一个不相交集合数据结构，每个顶点构成一个单独的集合
3. 按顺序考虑每条边，如果边的两个端点属于不同的集合，则将它们合并，并将该边加入最小生成树
4. 重复步骤3直到生成树包含n-1条边

#### 8.1.2 复杂度分析

- 排序边的时间复杂度：O(E log E)
- 不相交集合操作的时间复杂度：O(E α(V))
- 总时间复杂度：O(E log E)

### 8.2 连通分量检测

在无向图中检测连通分量：

```cpp
DisjointSet ds(numVertices);

// 对于图中的每条边(u,v)，合并u和v所在的集合
for (auto& edge : edges) {
    ds.unionSets(edge.u, edge.v);
}

// 统计连通分量的数量
int components = 0;
for (int i = 0; i < numVertices; i++) {
    if (ds.findSet(i) == i) {  // 如果i是根节点
        components++;
    }
}
```

### 8.3 图像处理中的连通区域标记

在图像处理中，可以使用不相交集合来标记连通区域：

```cpp
// 假设image是一个二维布尔数组，true表示前景像素
DisjointSet ds(width * height);

for (int y = 0; y < height; y++) {
    for (int x = 0; x < width; x++) {
        if (image[y][x]) {
            int current = y * width + x;
            
            // 检查上方像素
            if (y > 0 && image[y-1][x]) {
                int above = (y-1) * width + x;
                ds.unionSets(current, above);
            }
            
            // 检查左方像素
            if (x > 0 && image[y][x-1]) {
                int left = y * width + (x-1);
                ds.unionSets(current, left);
            }
        }
    }
}
```

## 📐 9. 数学分析

### 9.1 树的高度上界

通过按秩合并，我们可以证明以下定理：

**定理**：在按秩合并的不相交集合中，含有n个节点的树的高度至多为⌊log₂(n)⌋。

**证明**：
我们通过归纳法证明更强的结论：秩为k的节点，其子树至少包含2^k个节点。

基础情况：秩为0的节点是叶子节点，包含1=2^0个节点。

归纳步骤：假设所有秩小于k的节点满足条件。考虑一个秩为k的节点x，它是在两个秩为k-1的树合并时形成的。因此，x的子树包含至少2^(k-1) + 2^(k-1) = 2^k个节点。

### 9.2 路径压缩的摊还分析

路径压缩的摊还分析较为复杂，需要用到势能方法(potential method)。定义势函数Φ为所有节点的秩与该节点到其根节点的距离乘积之和。

通过精细的数学分析，可以证明使用按秩合并和路径压缩的不相交集合，m次操作的总时间复杂度为O(m·α(n))。

## ⚠️ 10. 实现注意事项

### 10.1 初始化

确保在使用前正确初始化数据结构，每个元素都应该初始化为自己集合的代表。

### 10.2 边界条件

注意处理以下特殊情况：
- 对不存在的元素执行操作
- 对已经在一个集合中的元素执行UNION操作
- 空集合的处理

### 10.3 内存管理

在动态环境中，要注意及时释放不再使用的内存。

### 10.4 线程安全

标准的不相交集合实现不是线程安全的。在多线程环境中使用时，需要添加适当的同步机制。
