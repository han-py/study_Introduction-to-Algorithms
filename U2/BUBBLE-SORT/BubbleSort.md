# 冒泡排序 (Bubble Sort)

## 1. 算法简介

冒泡排序（Bubble Sort）是一种简单的排序算法，它重复地遍历要排序的数列，一次比较两个元素，如果它们的顺序错误就把它们交换过来。遍历数列的工作是重复地进行直到没有再需要交换，也就是说该数列已经排序完成。

这个算法的名字由来是因为越小的元素会经由交换慢慢"浮"到数列的顶端（就像气泡一样逐渐上升到水面），所以被称为"冒泡排序"。

## 2. 算法思想

冒泡排序的基本思想是：

1. 比较相邻的元素。如果第一个比第二个大，就交换他们两个。
2. 对每一对相邻元素做同样的工作，从开始第一对到结尾的最后一对。这步做完后，最后的元素会是最大的数。
3. 针对所有的元素重复以上的步骤，除了最后一个。
4. 持续每次对越来越少的元素重复上面的步骤，直到没有任何一对数字需要比较。

## 3. 算法步骤

冒泡排序算法的具体步骤如下：

1. 比较相邻的两个元素，如果前面的元素大于后面的元素，则交换这两个元素。
2. 对每一对相邻元素做同样的工作，从第一对到最后一对，完成后最后的元素将是最大的元素。
3. 忽略上一步中找到的最大元素，重复以上步骤，直到没有任何一对数字需要比较。
4. 如果某一轮遍历中没有发生任何交换，则表示数组已经有序，可以提前结束。

## 4. 算法图解示例

以数组 [5, 3, 4, 1, 2] 为例：

| 轮次 | 数组状态 | 说明 |
|------|----------|------|
| 初始 | [5, 3, 4, 1, 2] | 原始数组 |
| 第1轮 | [3, 4, 1, 2, 5] | 5 移动到末尾 |
| 第2轮 | [3, 1, 2, 4, 5] | 4 移动到倒数第二位 |
| 第3轮 | [1, 2, 3, 4, 5] | 3 移动到正确位置 |
| 第4轮 | [1, 2, 3, 4, 5] | 没有变化，排序完成 |

每一轮的具体比较过程：
- 第1轮：
  - 比较 5 和 3，交换得到 [3, 5, 4, 1, 2]
  - 比较 5 和 4，交换得到 [3, 4, 5, 1, 2]
  - 比较 5 和 1，交换得到 [3, 4, 1, 5, 2]
  - 比较 5 和 2，交换得到 [3, 4, 1, 2, 5]

## 5. 伪代码实现

```
BUBBLE-SORT(A)
for i = 1 to A.length-1
    swapped = false
    for j = 1 to A.length-i
        if A[j] > A[j+1]
            swap A[j] and A[j+1]
            swapped = true
    if not swapped
        break
```

## 6. C++代码实现

```cpp
#include <iostream>
#include <vector>

void bubbleSort(std::vector<int>& arr) {
    int n = arr.size();
    for (int i = 0; i < n - 1; i++) {
        bool swapped = false;
        // 每轮将最大元素"冒泡"到末尾
        for (int j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                // 交换元素
                std::swap(arr[j], arr[j + 1]);
                swapped = true;
            }
        }
        // 如果没有发生交换，说明数组已经有序
        if (!swapped) {
            break;
        }
    }
}

// 打印数组
void printArray(const std::vector<int>& arr) {
    for (int i = 0; i < arr.size(); i++) {
        std::cout << arr[i] << " ";
    }
    std::cout << std::endl;
}

// 测试示例
int main() {
    std::vector<int> arr = {5, 2, 4, 6, 1, 3};
    std::cout << "排序前: ";
    printArray(arr);
    
    bubbleSort(arr);
    
    std::cout << "排序后: ";
    printArray(arr);
    
    return 0;
}
```

## 7. 算法分析

### 时间复杂度

- **最好情况**：数组已经有序，时间复杂度为 O(n)
  - 此时只需要进行一轮遍历，发现没有元素需要交换就结束
  
- **最坏情况**：数组完全逆序，时间复杂度为 O(n²)
  - 此时需要进行 n(n-1)/2 次比较和交换操作
  
- **平均情况**：时间复杂度为 O(n²)
  - 平均情况下，需要进行约 n²/4 次交换和 n(n-1)/2 次比较

### 空间复杂度

- 冒泡排序是原地排序算法，空间复杂度为 O(1)
- 只需要常数级别的额外空间存储临时变量

### 稳定性

- 冒泡排序是一种稳定的排序算法
- 相等元素的相对位置在排序后不会改变（只交换相邻且逆序的元素）

## 8. 算法特点

### 优点

1. 实现简单，代码容易理解
2. 是稳定排序算法
3. 是原地排序算法（空间复杂度为 O(1)）
4. 具有自适应性，对于已经基本有序的数据效率较高
5. 可以检测数组是否已经有序，并在适当时候提前结束

### 缺点

1. 时间复杂度较高，为 O(n²)
2. 效率低于插入排序等其他简单排序算法
3. 对于大规模数据排序效率很低

## 9. 适用场景

1. **教学演示**：由于算法逻辑简单清晰，适合用于算法入门教学
2. **小规模数据集**：当数据量很小时，实现简单比效率更重要
3. **基本有序的数据集**：对于已经基本有序的数据，冒泡排序效率较高
4. **对稳定性有要求的场景**：当需要保持相等元素的原有顺序时

## 10. 与其他排序算法的比较

| 排序算法 | 最好时间复杂度 | 平均时间复杂度 | 最坏时间复杂度 | 空间复杂度 | 稳定性 |
|---------|---------------|---------------|---------------|-----------|--------|
| 冒泡排序 | O(n) | O(n²) | O(n²) | O(1) | 稳定 |
| 插入排序 | O(n) | O(n²) | O(n²) | O(1) | 稳定 |
| 选择排序 | O(n²) | O(n²) | O(n²) | O(1) | 不稳定 |
| 快速排序 | O(n log n) | O(n log n) | O(n²) | O(log n) | 不稳定 |
| 归并排序 | O(n log n) | O(n log n) | O(n log n) | O(n) | 稳定 |

## 11. 优化方案

虽然冒泡排序本身效率不高，但可以通过以下方式进行优化：

1. **提前终止**：如果某一轮没有发生任何交换，说明数组已经有序，可以提前结束
2. **记录最后交换位置**：记录最后一次交换的位置，下次比较只需要到该位置即可
3. **双向冒泡排序**：也称鸡尾酒排序，交替进行正向和反向遍历

## 12. 总结

冒泡排序是最简单的排序算法之一，它的名字来源于较小的元素会像气泡一样逐渐"浮"到数组的前端。虽然冒泡排序的时间复杂度较高，不适合处理大规模数据，但由于其实现简单、易于理解的特点，在计算机科学的教学中经常被用来介绍排序算法的基本概念。

在实际应用中，冒泡排序很少被使用，因为它的效率不如其他 O(n²) 算法（如插入排序）。但对于非常小的数据集或者几乎已经排好序的数据，冒泡排序仍然是一个可行的选择。
